## 面试流程

### 1. 介绍自己做过最有技术深度的一件事 —— Flutter Web 移植工程

- Flutter 部分的 UI 是怎么写的？是把客户端的 Kotlin 代码翻译过来吗？

- 你认为 Flutter 和 Android 原生开发有什么区别？  
答：讲到了命令式和声明式的 UI 开发，提到最近还在学习另外一种声明式框架 Jetpack Compose.

- 你认为命令式和声明式 UI 开发在编程思想方面有哪些区别呢？Flutter 和 Jetpack Compose 你认为哪种框架更好用？

- 简历上提到了 WebAssembly，简单介绍一下它？它是如何实现 JavaScript 和 C++ 层数据互通的？  
答：讲到 WASM 只能支持 Number 和 String，数据交换比较困难。

- 你们是怎么做数据互通的呢？为什么会选择这种方式？  
答：交换的是 JSON 字符串，为了做 MVP 产品，提高开发效率，选择了这种比较灵活的方式。

- 你还知道其他多语言数据互通的工具吗？  
答：知道 Protobuf，项目里正在用。

- Protobuf 和 JSON 有什么不同？对比一下这两者。

### 3. 做过几个中大型图像编辑类的需求开发，简单说说？

- 介绍了人像打光功能的实现方式，里面用到了很多手势处理

- 说说你是如何处理各种触控手势的，如果遇到手势冲突的情况应该怎么办？

- 项目中使用的是什么架构，用了哪些框架？  
答：架构用的是 MVVM，框架和库用 LiveData、DataBinding 比较多。

- LiveData 遇到数据重放的问题怎么办？  
答：工作场景比较少遇到，自学 Jetpack Compose 发现可以用 StateFlow 一类的流来解决。

- 流式数据结构，如果流速过快要怎么办？比如 MVI 架构中，事件发送的速度过快，来不及处理。
